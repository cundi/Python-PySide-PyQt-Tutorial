Detailed Description¶

The PySide.QtXml.QDomEntity class represents an XML entity.

This class represents an entity in an XML document, either parsed or unparsed. Note that this models the entity itself not the entity declaration.

DOM does not support editing entity nodes; if a user wants to make changes to the contents of an entity, every related PySide.QtXml.QDomEntityReference node must be replaced in the DOM tree by a clone of the entity’s contents, and then the desired changes must be made to each of the clones instead. All the descendants of an entity node are read-only.

An entity node does not have any parent.

You can access the entity’s PySide.QtXml.QDomEntity.publicId() , PySide.QtXml.QDomEntity.systemId() and PySide.QtXml.QDomEntity.notationName() when available.

For further information about the Document Object Model see Level 1 and Level 2 Core. For a more general introduction of the DOM implementation see the PySide.QtXml.QDomDocument documentation.

Constructs a copy of x .

The data of the copy is shared (shallow copy): modifying one node will also change the other. If you want to make a deep copy, use PySide.QtXml.QDomNode.cloneNode() .

PySide.QtXml.QDomEntity.notationName()¶
Return type:unicode

For unparsed entities this function returns the name of the notation for the entity. For parsed entities this function returns an empty string.

PySide.QtXml.QDomEntity.publicId()¶
Return type:unicode

Returns the public identifier associated with this entity. If the public identifier was not specified an empty string is returned.

PySide.QtXml.QDomEntity.systemId()¶
Return type:unicode

Returns the system identifier associated with this entity. If the system identifier was not specified an empty string is returned.
