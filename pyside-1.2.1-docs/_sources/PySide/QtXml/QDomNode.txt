Detailed Description
--------------------



    The :class:`PySide.QtXml.QDomNode` class is the base class for all the nodes in a DOM tree.

    Many functions in the DOM return a :class:`PySide.QtXml.QDomNode` .

    You can find out the type of a node using :meth:`PySide.QtXml.QDomNode.isAttr` , :meth:`PySide.QtXml.QDomNode.isCDATASection` , :meth:`PySide.QtXml.QDomNode.isDocumentFragment` , :meth:`PySide.QtXml.QDomNode.isDocument` , :meth:`PySide.QtXml.QDomNode.isDocumentType` , :meth:`PySide.QtXml.QDomNode.isElement` , :meth:`PySide.QtXml.QDomNode.isEntityReference` , :meth:`PySide.QtXml.QDomNode.isText` , :meth:`PySide.QtXml.QDomNode.isEntity` , :meth:`PySide.QtXml.QDomNode.isNotation` , :meth:`PySide.QtXml.QDomNode.isProcessingInstruction` , :meth:`PySide.QtXml.QDomNode.isCharacterData` and :meth:`PySide.QtXml.QDomNode.isComment` .

    A :class:`PySide.QtXml.QDomNode` can be converted into one of its subclasses using :meth:`PySide.QtXml.QDomNode.toAttr` , :meth:`PySide.QtXml.QDomNode.toCDATASection` , :meth:`PySide.QtXml.QDomNode.toDocumentFragment` , :meth:`PySide.QtXml.QDomNode.toDocument` , :meth:`PySide.QtXml.QDomNode.toDocumentType` , :meth:`PySide.QtXml.QDomNode.toElement` , :meth:`PySide.QtXml.QDomNode.toEntityReference` , :meth:`PySide.QtXml.QDomNode.toText` , :meth:`PySide.QtXml.QDomNode.toEntity` , :meth:`PySide.QtXml.QDomNode.toNotation` , :meth:`PySide.QtXml.QDomNode.toProcessingInstruction` , :meth:`PySide.QtXml.QDomNode.toCharacterData` or :meth:`PySide.QtXml.QDomNode.toComment` . You can convert a node to a null node with :meth:`PySide.QtXml.QDomNode.clear` .

    Copies of the :class:`PySide.QtXml.QDomNode` class share their data using explicit sharing. This means that modifying one node will change all copies. This is especially useful in combination with functions which return a :class:`PySide.QtXml.QDomNode` , e.g. :meth:`PySide.QtXml.QDomNode.firstChild` . You can make an independent (deep) copy of the node with :meth:`PySide.QtXml.QDomNode.cloneNode` .

    A :class:`PySide.QtXml.QDomNode` can be null, much like a null pointer. Creating a copy of a null node results in another null node. It is not possible to modify a null node, but it is possible to assign another, possibly non-null node to it. In this case, the copy of the null node will remain null. You can check if a :class:`PySide.QtXml.QDomNode` is null by calling :meth:`PySide.QtXml.QDomNode.isNull` . The empty constructor of a :class:`PySide.QtXml.QDomNode` (or any of the derived classes) creates a null node.

    Nodes are inserted with :meth:`PySide.QtXml.QDomNode.insertBefore` , :meth:`PySide.QtXml.QDomNode.insertAfter` or :meth:`PySide.QtXml.QDomNode.appendChild` . You can replace one node with another using :meth:`PySide.QtXml.QDomNode.replaceChild` and remove a node with :meth:`PySide.QtXml.QDomNode.removeChild` .

    To traverse nodes use :meth:`PySide.QtXml.QDomNode.firstChild` to get a node's first child (if any), and :meth:`PySide.QtXml.QDomNode.nextSibling` to traverse. :class:`PySide.QtXml.QDomNode` also provides :meth:`PySide.QtXml.QDomNode.lastChild` , :meth:`PySide.QtXml.QDomNode.previousSibling` and :meth:`PySide.QtXml.QDomNode.parentNode` . To find the first child node with a particular node name use :meth:`PySide.QtXml.QDomNode.namedItem` .

    To find out if a node has children use :meth:`PySide.QtXml.QDomNode.hasChildNodes` and to get a list of all of a node's children use :meth:`PySide.QtXml.QDomNode.childNodes` .

    The node's name and value (the meaning of which varies depending on its type) is returned by :meth:`PySide.QtXml.QDomNode.nodeName` and :meth:`PySide.QtXml.QDomNode.nodeValue` respectively. The node's type is returned by :meth:`PySide.QtXml.QDomNode.nodeType` . The node's value can be set with :meth:`PySide.QtXml.QDomNode.setNodeValue` .

    The document to which the node belongs is returned by :meth:`PySide.QtXml.QDomNode.ownerDocument` .

    Adjacent :class:`PySide.QtXml.QDomText` nodes can be merged into a single node with :meth:`PySide.QtXml.QDomNode.normalize` .

    :class:`PySide.QtXml.QDomElement` nodes have attributes which can be retrieved with :meth:`PySide.QtXml.QDomNode.attributes` .

    :class:`PySide.QtXml.QDomElement` and :class:`PySide.QtXml.QDomAttr` nodes can have namespaces which can be retrieved with :meth:`PySide.QtXml.QDomNode.namespaceURI` . Their local name is retrieved with :meth:`PySide.QtXml.QDomNode.localName` , and their prefix with :meth:`PySide.QtXml.QDomNode.prefix` . The prefix can be set with :meth:`PySide.QtXml.QDomNode.setPrefix` .

    You can write the XML representation of the node to a text stream with :meth:`PySide.QtXml.QDomNode.save` .

    The following example looks for the first element in an XML document and prints the names of all the elements that are its direct children.

    ::

        d = QDomDocument()
        d.setContent(someXML)
        n = d.firstChild()
        while !n.isNull():
            if n.isElement():
                e = n.toElement()
                print "Element name: %s" % e.tagName()
                break
            n = n.nextSibling()


    For further information about the Document Object Model see Level 1 and Level 2 Core. For a more general introduction of the DOM implementation see the :class:`PySide.QtXml.QDomDocument` documentation.

.. method:: PySide.QtXml.QDomNode.appendChild(newChild)


    :param newChild: :class:`PySide.QtXml.QDomNode`
    :rtype: :class:`PySide.QtXml.QDomNode`




    Appends ``newChild`` as the node's last child.

    If ``newChild`` is the child of another node, it is reparented to this node. If ``newChild`` is a child of this node, then its position in the list of children is changed.

    If ``newChild`` is a :class:`PySide.QtXml.QDomDocumentFragment` , then the children of the fragment are removed from the fragment and appended.

    If ``newChild`` is a :class:`PySide.QtXml.QDomElement` and this node is a :class:`PySide.QtXml.QDomDocument` that already has an element node as a child, ``newChild`` is not added as a child and a null node is returned.

    Returns a new reference to ``newChild`` on success or a :meth:`null node<PySide.QtXml.QDomNode.isNull>` on failure.

    Calling this function on a null node(created, for example, with the default constructor) does nothing and returns a :meth:`null node<PySide.QtXml.QDomNode.isNull>` .

    The DOM specification disallow inserting attribute nodes, but for historical reasons, QDom accepts them anyway.

    .. seealso::  :meth:`PySide.QtXml.QDomNode.insertBefore`  :meth:`PySide.QtXml.QDomNode.insertAfter`  :meth:`PySide.QtXml.QDomNode.replaceChild`  :meth:`PySide.QtXml.QDomNode.removeChild`

.. method:: PySide.QtXml.QDomNode.childNodes()


    :rtype: :class:`PySide.QtXml.QDomNodeList`




    Returns a list of all direct child nodes.

    Most often you will call this function on a :class:`PySide.QtXml.QDomElement` object.

    For example, if the XML document looks like this:

    ::

        <body>
        <h1>Heading</h1>
        <p>Hello <b>you</b></p>
        </body>


    Then the list of child nodes for the "body"-element will contain the node created by the &lt;h1&gt; tag and the node created by the &lt;p&gt; tag.

    The nodes in the list are not copied; so changing the nodes in the list will also change the children of this node.

    .. seealso::  :meth:`PySide.QtXml.QDomNode.firstChild`  :meth:`PySide.QtXml.QDomNode.lastChild`

.. method:: PySide.QtXml.QDomNode.cloneNode([deep=true])


    :param deep: :class:`PySide.QtCore.bool`
    :rtype: :class:`PySide.QtXml.QDomNode`




    Creates a deep (not shallow) copy of the :class:`PySide.QtXml.QDomNode` .

    If ``deep`` is true, then the cloning is done recursively which means that all the node's children are deep copied too. If ``deep`` is false only the node itself is copied and the copy will have no child nodes.

.. method:: PySide.QtXml.QDomNode.insertAfter(newChild, refChild)


    :param newChild: :class:`PySide.QtXml.QDomNode`
    :param refChild: :class:`PySide.QtXml.QDomNode`
    :rtype: :class:`PySide.QtXml.QDomNode`




    Inserts the node ``newChild`` after the child node ``refChild`` . ``refChild`` must be a direct child of this node. If ``refChild`` is :meth:`null<PySide.QtXml.QDomNode.isNull>` then ``newChild`` is appended as this node's last child.

    If ``newChild`` is the child of another node, it is reparented to this node. If ``newChild`` is a child of this node, then its position in the list of children is changed.

    If ``newChild`` is a :class:`PySide.QtXml.QDomDocumentFragment` , then the children of the fragment are removed from the fragment and inserted after ``refChild`` .

    Returns a new reference to ``newChild`` on success or a :meth:`null node<PySide.QtXml.QDomNode.isNull>` on failure.

    The DOM specification disallow inserting attribute nodes, but due to historical reasons QDom accept them nevertheless.

.. method:: PySide.QtXml.QDomNode.__eq__(arg__1)


    :param arg__1: :class:`PySide.QtXml.QDomNode`
    :rtype: :class:`PySide.QtCore.bool`




    Returns true if ``n`` and this DOM node are equal; otherwise returns false.

    Any instance of :class:`PySide.QtXml.QDomNode` acts as a reference to an underlying data structure in :class:`PySide.QtXml.QDomDocument` . The test for equality checks if the two references point to the same underlying node. For example:

    ::

        QDomDocument document
        QDomElement element1 = document.documentElement()
        QDomElement element2 = element1


    The two nodes ( :class:`PySide.QtXml.QDomElement` is a :class:`PySide.QtXml.QDomNode` subclass) both refer to the document's root element, and ``element1 == element2`` will return true. On the other hand:

    ::

        QDomElement element3 = document.createElement("MyElement")
        QDomElement element4 = document.createElement("MyElement")


    Even though both nodes are empty elements carrying the same name, ``element3 == element4`` will return false because they refer to two different nodes in the underlying data structure.
