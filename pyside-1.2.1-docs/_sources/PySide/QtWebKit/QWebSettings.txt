Detailed Description
--------------------



    The :class:`PySide.QtWebKit.QWebSettings` class provides an object to store the settings used by :class:`PySide.QtWebKit.QWebPage` and :class:`PySide.QtWebKit.QWebFrame` .

    Each :class:`PySide.QtWebKit.QWebPage` object has its own :class:`PySide.QtWebKit.QWebSettings` object, which configures the settings for that page. If a setting is not configured, then it is looked up in the global settings object, which can be accessed using :meth:`PySide.QtWebKit.QWebSettings.globalSettings` .

    :class:`PySide.QtWebKit.QWebSettings` allows configuration of browser properties, such as font sizes and families, the location of a custom style sheet, and generic attributes like JavaScript and plugins. Individual attributes are set using the :meth:`PySide.QtWebKit.QWebSettings.setAttribute` function. The :attr:`QWebSettings.WebAttribute` enum further describes each attribute.

    :class:`PySide.QtWebKit.QWebSettings` also configures global properties such as the web page memory cache, icon database, local database storage and offline applications storage.

    .. _PySide.QtWebKit.QWebSettings_enabling-plugins:



Enabling Plugins
^^^^^^^^^^^^^^^^

    Support for browser plugins can enabled by setting the :attr:`PluginsEnabled<QWebSettings.WebAttribute>` attribute. For many applications, this attribute is enabled for all pages by setting it on the :meth:`global settings object<PySide.QtWebKit.QWebSettings.globalSettings>` . :ref:`QtWebKit` will always ignore this setting when processing Qt plugins. The decision to allow a Qt plugin is made by the client in its reimplementation of :meth:`QWebPage.createPlugin()<PySide.QtWebKit.QWebPage.createPlugin>` .

    .. _PySide.QtWebKit.QWebSettings_web-application-support:



Web Application Support
^^^^^^^^^^^^^^^^^^^^^^^

    :ref:`WebKit<webkit.html>` provides support for features specified in :ref:`HTML 5<http://www.w3.org/TR/html5/>` that improve the performance and capabilities of Web applications. These include client-side (offline) storage and the use of a Web application cache.

    Client-side (offline) storage is an improvement over the use of cookies to store persistent data in Web applications. Applications can configure and enable the use of an offline storage database by calling the :meth:`PySide.QtWebKit.QWebSettings.setOfflineStoragePath` with an appropriate file path, and can limit the quota for each application by calling :meth:`PySide.QtWebKit.QWebSettings.setOfflineStorageDefaultQuota` .

    .. seealso::  :meth:`QWebPage.settings()<PySide.QtWebKit.QWebPage.settings>`  :meth:`QWebView.settings()<PySide.QtWebKit.QWebView.settings>`  :ref:`Web Browser<demos-browser.html>`

.. staticmethod:: PySide.QtWebKit.QWebSettings.setObjectCacheCapacities(cacheMinDeadCapacity, cacheMaxDead, totalCapacity)


    :param cacheMinDeadCapacity: :class:`PySide.QtCore.int`
    :param cacheMaxDead: :class:`PySide.QtCore.int`
    :param totalCapacity: :class:`PySide.QtCore.int`




    Specifies the capacities for the memory cache for dead objects such as stylesheets or scripts.

    The ``cacheMinDeadCapacity`` specifies the *minimum* number of bytes that dead objects should consume when the cache is under pressure.

    ``cacheMaxDead`` is the *maximum* number of bytes that dead objects should consume when the cache is **not** under pressure.

    ``totalCapacity`` specifies the *maximum* number of bytes that the cache should consume **overall** .

    The cache is enabled by default. Calling setObjectCacheCapacities(0, 0, 0) will disable the cache. Calling it with one non-zero enables it again.
