Detailed Description
--------------------



    The :class:`PySide.QtCore.QEvent` class is the base class of all event classes. Event objects contain event parameters.

    Qt's main event loop ( :meth:`QCoreApplication.exec()<PySide.QtCore.QCoreApplication.exec>` ) fetches native window system events from the event queue, translates them into QEvents, and sends the translated events to :class:`PySide.QtCore.QObject` s.

    In general, events come from the underlying window system ( :meth:`PySide.QtCore.QEvent.spontaneous` returns true), but it is also possible to manually send events using :meth:`QCoreApplication.sendEvent()<PySide.QtCore.QCoreApplication.sendEvent>` and :meth:`QCoreApplication.postEvent()<PySide.QtCore.QCoreApplication.postEvent>` ( :meth:`PySide.QtCore.QEvent.spontaneous` returns false).

    :ref:`QObjects<qtwebkit-bridge.html>` receive events by having their :meth:`QObject.event()<PySide.QtCore.QObject.event>` function called. The function can be reimplemented in subclasses to customize event handling and add additional event types; :meth:`QWidget.event()<PySide.QtGui.QWidget.event>` is a notable example. By default, events are dispatched to event handlers like :meth:`QObject.timerEvent()<PySide.QtCore.QObject.timerEvent>` and :meth:`QWidget.mouseMoveEvent()<PySide.QtGui.QWidget.mouseMoveEvent>` . :meth:`QObject.installEventFilter()<PySide.QtCore.QObject.installEventFilter>` allows an object to intercept events destined for another object.

    The basic :class:`PySide.QtCore.QEvent` contains only an event type parameter and an "accept" flag. The accept flag set with :meth:`PySide.QtCore.QEvent.accept` , and cleared with :meth:`PySide.QtCore.QEvent.ignore` . It is set by default, but don't rely on this as subclasses may choose to clear it in their constructor.

    Subclasses of :class:`PySide.QtCore.QEvent` contain additional parameters that describe the particular event.

    .. seealso::  :meth:`QObject.event()<PySide.QtCore.QObject.event>`  :meth:`QObject.installEventFilter()<PySide.QtCore.QObject.installEventFilter>`  :meth:`QWidget.event()<PySide.QtGui.QWidget.event>`  :meth:`QCoreApplication.sendEvent()<PySide.QtCore.QCoreApplication.sendEvent>`  :meth:`QCoreApplication.postEvent()<PySide.QtCore.QCoreApplication.postEvent>`  :meth:`QCoreApplication.processEvents()<PySide.QtCore.QCoreApplication.processEvents>`

.. method:: PySide.QtCore.QEvent.accept()






    Sets the accept flag of the event object, the equivalent of calling setAccepted(true).

    Setting the accept parameter indicates that the event receiver wants the event. Unwanted events might be propagated to the parent widget.

    .. seealso::  :meth:`PySide.QtCore.QEvent.ignore` 


.. method:: PySide.QtCore.QEvent.ignore()






    Clears the accept flag parameter of the event object, the equivalent of calling setAccepted(false).

    Clearing the accept parameter indicates that the event receiver does not want the event. Unwanted events might be propagated to the parent widget.

    .. seealso::  :meth:`PySide.QtCore.QEvent.accept`

    :rtype: :class:`PySide.QtCore.bool`






.. staticmethod:: PySide.QtCore.QEvent.registerEventType([hint=-1])


    :param hint: :class:`PySide.QtCore.int`
    :rtype: :class:`PySide.QtCore.int`




    Registers and returns a custom event type. The ``hint`` provided will be used if it is available, otherwise it will return a value between :attr:`QEvent.User<QEvent.Type>` and :attr:`QEvent.MaxUser<QEvent.Type>` that has not yet been registered. The ``hint`` is ignored if its value is not between :attr:`QEvent.User<QEvent.Type>` and :attr:`QEvent.MaxUser<QEvent.Type>` .

.. method:: PySide.QtCore.QEvent.spontaneous()


    :rtype: :class:`PySide.QtCore.bool`




    Returns true if the event originated outside the application (a system event); otherwise returns false.

    The return value of this function is not defined for paint events.



.. method:: PySide.QtCore.QEvent.type()


    :rtype: :attr:`PySide.QtCore.QEvent.Type`




    Returns the event type.
