Detailed Description
--------------------



    The :class:`PySide.QtGui.QGraphicsWidget` class is the base class for all widget items in a :class:`PySide.QtGui.QGraphicsScene` .

    :class:`PySide.QtGui.QGraphicsWidget` is an extended base item that provides extra functionality over :class:`PySide.QtGui.QGraphicsItem` . It is similar to :class:`PySide.QtGui.QWidget` in many ways:


    * Provides a :meth:`PySide.QtGui.QGraphicsWidget.palette` , a :meth:`PySide.QtGui.QGraphicsWidget.font` and a :meth:`PySide.QtGui.QGraphicsWidget.style` .
    * Has a defined :meth:`PySide.QtGui.QGraphicsLayoutItem.geometry` .
    * Supports layouts with :meth:`PySide.QtGui.QGraphicsWidget.setLayout` and :meth:`PySide.QtGui.QGraphicsWidget.layout` .
    * Supports shortcuts and actions with :meth:`PySide.QtGui.QGraphicsWidget.grabShortcut` and :meth:`PySide.QtGui.QGraphicsWidget.insertAction`

    Unlike :class:`PySide.QtGui.QGraphicsItem` , :class:`PySide.QtGui.QGraphicsWidget` is not an abstract class; you can create instances of a :class:`PySide.QtGui.QGraphicsWidget` without having to subclass it. This approach is useful for widgets that only serve the purpose of organizing child widgets into a layout.

    :class:`PySide.QtGui.QGraphicsWidget` can be used as a base item for your own custom item if you require advanced input focus handling, e.g., tab focus and activation, or layouts.

    Since :class:`PySide.QtGui.QGraphicsWidget` resembles :class:`PySide.QtGui.QWidget` and has similar API, it is easier to port a widget from :class:`PySide.QtGui.QWidget` to :class:`PySide.QtGui.QGraphicsWidget` , instead of :class:`PySide.QtGui.QGraphicsItem` .

    .. note:: \ :class:`PySide.QtGui.QWidget` -based widgets can be directly embedded into a :class:`PySide.QtGui.QGraphicsScene` using :class:`PySide.QtGui.QGraphicsProxyWidget` .

    Noticeable differences between :class:`PySide.QtGui.QGraphicsWidget` and :class:`PySide.QtGui.QWidget` are:

    +--------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------+
    |:class:`PySide.QtGui.QGraphicsWidget`                                                                                           |:class:`PySide.QtGui.QWidget`                                                                                        |
    +--------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------+
    |Coordinates and geometry are defined with qreals (doubles or floats, depending on the platform).                                |:class:`PySide.QtGui.QWidget` uses integer geometry ( :class:`PySide.QtCore.QPoint` , :class:`PySide.QtCore.QRect` ).|
    +--------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------+
    |The widget is already visible by default; you do not have to call :meth:`PySide.QtGui.QGraphicsItem.show` to display the widget.|:class:`PySide.QtGui.QWidget` is hidden by default until you call :meth:`PySide.QtGui.QGraphicsItem.show` .          |
    +--------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------+
    |A subset of widget attributes are supported.                                                                                    |All widget attributes are supported.                                                                                 |
    +--------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------+
    |A top-level item's style defaults to QGraphicsScene::style                                                                      |A top-level widget's style defaults to QApplication::style                                                           |
    +--------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------+
    |Graphics View provides a custom drag and drop framework, different from :class:`PySide.QtGui.QWidget` .                         |Standard drag and drop framework.                                                                                    |
    +--------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------+
    |Widget items do not support modality.                                                                                           |Full modality support.                                                                                               |
    +--------------------------------------------------------------------------------------------------------------------------------+---------------------------------------------------------------------------------------------------------------------+

    :class:`PySide.QtGui.QGraphicsWidget` supports a subset of Qt's widget attributes, ( :attr:`Qt.WidgetAttribute` ), as shown in the table below. Any attributes not listed in this table are unsupported, or otherwise unused.

    +----------------------------------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+

    |:attr:`Qt.WA_SetLayoutDirection<Qt.WidgetAttribute>`|Set by :meth:`PySide.QtGui.QGraphicsWidget.setLayoutDirection` , cleared by :meth:`PySide.QtGui.QGraphicsWidget.unsetLayoutDirection` .     +----------------------------------------------------+------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------+
    |:attr:`Qt.WA_RightToLeft<Qt.WidgetAttribute>`       |Toggled by :meth:`PySide.QtGui.QGraphicsWidget.setLayoutDirection` . Inherited from the parent/scene.     Although :class:`PySide.QtGui.QGraphicsWidget` inherits from both :class:`PySide.QtCore.QObject` and :class:`PySide.QtGui.QGraphicsItem` , you should use the functions provided by :class:`PySide.QtGui.QGraphicsItem` , *not*\ :class:`PySide.QtCore.QObject` , to manage the relationships between parent and child items. These functions control the stacking order of items as well as their ownership.

    .. note::  The :meth:`QObject.parent()<PySide.QtCore.QObject.parent>` should always return 0 for QGraphicsWidgets, but this policy is not strictly defined.

.. method:: PySide.QtGui.QGraphicsWidget.autoFillBackground()


    :rtype: :class:`PySide.QtCore.bool`




    This property holds whether the widget background is filled automatically.

    If enabled, this property will cause Qt to fill the background of the widget before invoking the :meth:`PySide.QtGui.QGraphicsWidget.paint` method. The color used is defined by the :attr:`QPalette.Window<QPalette.ColorRole>` color role from the widget's :class:`palette<PySide.QtGui.QPalette>` .

    In addition, Windows are always filled with :attr:`QPalette.Window<QPalette.ColorRole>` , unless the WA\_OpaquePaintEvent or WA\_NoSystemBackground attributes are set.

    By default, this property is false.

    .. seealso::  :attr:`Qt.WA_OpaquePaintEvent<Qt.WidgetAttribute>`  :attr:`Qt.WA_NoSystemBackground<Qt.WidgetAttribute>`

.. method:: PySide.QtGui.QGraphicsWidget.close()


    :rtype: :class:`PySide.QtCore.bool`




    Call this function to close the widget.

    Returns true if the widget was closed; otherwise returns false. This slot will first send a :class:`PySide.QtGui.QCloseEvent` to the widget, which may or may not accept the event. If the event was ignored, nothing happens. If the event was accepted, it will :meth:`PySide.QtGui.QGraphicsItem.hide` the widget.

    If the widget has the :attr:`Qt.WA_DeleteOnClose<Qt.WidgetAttribute>` attribute set it will be deleted.

.. method:: PySide.QtGui.QGraphicsWidget.focusNextPrevChild(next)


    :param next: :class:`PySide.QtCore.bool`
    :rtype: :class:`PySide.QtCore.bool`




    Finds a new widget to give the keyboard focus to, as appropriate for Tab and Shift+Tab, and returns true if it can find a new widget; returns false otherwise. If ``next`` is true, this function searches forward; if ``next`` is false, it searches backward.

    Sometimes, you will want to reimplement this function to provide special focus handling for your widget and its subwidgets. For example, a web browser might reimplement it to move its current active link forward or backward, and call the base implementation only when it reaches the last or first link on the page.

    Child widgets call :meth:`PySide.QtGui.QGraphicsWidget.focusNextPrevChild` on their parent widgets, but only the window that contains the child widgets decides where to redirect focus. By reimplementing this function for an object, you gain control of focus traversal for all child widgets.

    .. seealso::  :meth:`PySide.QtGui.QGraphicsWidget.focusPolicy` 


.. method:: PySide.QtGui.QGraphicsWidget.focusPolicy()


    :rtype: :attr:`PySide.QtCore.Qt.FocusPolicy`




    This property holds the way the widget accepts keyboard focus.

    The focus policy is :attr:`Qt.TabFocus<Qt.FocusPolicy>` if the widget accepts keyboard focus by tabbing, :attr:`Qt.ClickFocus<Qt.FocusPolicy>` if the widget accepts focus by clicking, :attr:`Qt.StrongFocus<Qt.FocusPolicy>` if it accepts both, and :attr:`Qt.NoFocus<Qt.FocusPolicy>` (the default) if it does not accept focus at all.

    You must enable keyboard focus for a widget if it processes keyboard events. This is normally done from the widget's constructor. For instance, the :class:`PySide.QtGui.QLineEdit` constructor calls setFocusPolicy( :attr:`Qt.StrongFocus<Qt.FocusPolicy>` ).

    If you enable a focus policy (i.e., not :attr:`Qt.NoFocus<Qt.FocusPolicy>` ), :class:`PySide.QtGui.QGraphicsWidget` will automatically enable the :attr:`ItemIsFocusable<QGraphicsItem.GraphicsItemFlag>` flag. Setting :attr:`Qt.NoFocus<Qt.FocusPolicy>` on a widget will clear the :attr:`ItemIsFocusable<QGraphicsItem.GraphicsItemFlag>` flag. If the widget currently has keyboard focus, the widget will automatically lose focus.

.. method:: PySide.QtGui.QGraphicsWidget.font()


    :rtype: :class:`PySide.QtGui.QFont`




    This property holds the widgets' font.

    This property provides the widget's font.

    :class:`PySide.QtGui.QFont` consists of font properties that have been explicitly defined and properties implicitly inherited from the widget's parent. Hence, :meth:`PySide.QtGui.QGraphicsWidget.font` can return a different font compared to the one set with :meth:`PySide.QtGui.QGraphicsWidget.setFont` . This scheme allows you to define single entries in a font without affecting the font's inherited entries.

    When a widget's font changes, it resolves its entries against its parent widget. If the widget does not have a parent widget, it resolves its entries against the scene. The widget then sends itself a :attr:`FontChange<QEvent.Type>` event and notifies all its descendants so that they can resolve their fonts as well.

    By default, this property contains the application's default font.

    .. seealso::  :meth:`QApplication.font()<PySide.QtGui.QApplication.font>`  :meth:`QGraphicsScene.font<PySide.QtGui.QGraphicsScene.font>`  :meth:`QFont.resolve()<PySide.QtGui.QFont.resolve>`

.. staticmethod:: PySide.QtGui.QGraphicsWidget.setTabOrder(first, second)


    :param first: :class:`PySide.QtGui.QGraphicsWidget`
    :param second: :class:`PySide.QtGui.QGraphicsWidget`




    Moves the ``second`` widget around the ring of focus widgets so that keyboard focus moves from the ``first`` widget to the ``second`` widget when the Tab key is pressed.

    Note that since the tab order of the ``second`` widget is changed, you should order a chain like this:

    ::

        setTabOrder(a, b) # a to b
        setTabOrder(b, c) # a to b to c
        setTabOrder(c, d) # a to b to c to d


    *not* like this:

    ::

        # WRONG
        setTabOrder(c, d) # c to d
        setTabOrder(a, b) # a to b AND c to d
        setTabOrder(b, c) # a to b to c, but not c to d


    If ``first`` is 0, this indicates that ``second`` should be the first widget to receive input focus should the scene gain Tab focus (i.e., the user hits Tab so that focus passes into the scene). If ``second`` is 0, this indicates that ``first`` should be the first widget to gain focus if the scene gained BackTab focus.

    By default, tab order is defined implicitly using widget creation order.

    Calling :meth:`PySide.QtGui.QGraphicsWidget.resize` resizes the widget to a ``size`` bounded by :meth:`PySide.QtGui.QGraphicsLayoutItem.minimumSize` and :meth:`PySide.QtGui.QGraphicsLayoutItem.maximumSize` . This property only affects the widget's width and height (e.g., its right and bottom edges); the widget's position and top-left corner remains unaffected.

    Resizing a widget triggers the widget to immediately receive a :attr:`GraphicsSceneResize<QEvent.Type>` event with the widget's old and new size. If the widget has a layout assigned when this event arrives, the layout will be activated and it will automatically update any child widgets's geometry.

    This property does not affect any layout of the parent widget. If the widget itself is managed by a parent layout; e.g., it has a parent widget with a layout assigned, that layout will not activate.

    By default, this property contains a size with zero width and height.

.. method:: PySide.QtGui.QGraphicsWidget.unsetLayoutDirection()






    This property holds the layout direction for this widget..

    This property modifies this widget's and all of its descendants' :attr:`Qt.WA_RightToLeft<Qt.WidgetAttribute>` attribute. It also sets this widget's :attr:`Qt.WA_SetLayoutDirection<Qt.WidgetAttribute>` attribute.

    The widget's layout direction determines the order in which the layout manager horizontally arranges subwidgets of this widget. The default value depends on the language and locale of the application, and is typically in the same direction as words are read and written. With :attr:`Qt.LeftToRight<Qt.LayoutDirection>` , the layout starts placing subwidgets from the left side of this widget towards the right. :attr:`Qt.RightToLeft<Qt.LayoutDirection>` does the opposite - the layout will place widgets starting from the right edge moving towards the left.

    Subwidgets inherit their layout direction from the parent. Top-level widget items inherit their layout direction from QGraphicsScene::layoutDirection. If you change a widget's layout direction by calling :meth:`PySide.QtGui.QGraphicsWidget.setLayoutDirection` , the widget will send itself a :attr:`LayoutDirectionChange<QEvent.Type>` event, and then propagate the new layout direction to all its descendants.

    .. seealso::  :meth:`QWidget.layoutDirection<PySide.QtGui.QWidget.layoutDirection>`  :meth:`QApplication.layoutDirection<PySide.QtGui.QApplication.layoutDirection>`
