Detailed Description
--------------------



    The :class:`PySide.QtGui.QGraphicsView` class provides a widget for displaying the contents of a :class:`PySide.QtGui.QGraphicsScene` .

    :class:`PySide.QtGui.QGraphicsView` visualizes the contents of a :class:`PySide.QtGui.QGraphicsScene` in a scrollable viewport. To create a scene with geometrical items, see :class:`PySide.QtGui.QGraphicsScene` 's documentation. :class:`PySide.QtGui.QGraphicsView` is part of the :ref:`Graphics View Framework<graphicsview.html>` .

    To visualize a scene, you start by constructing a :class:`PySide.QtGui.QGraphicsView` object, passing the address of the scene you want to visualize to :class:`PySide.QtGui.QGraphicsView` 's constructor. Alternatively, you can call :meth:`PySide.QtGui.QGraphicsView.setScene` to set the scene at a later point. After you call :meth:`PySide.QtGui.QWidget.show` , the view will by default scroll to the center of the scene and display any items that are visible at this point. For example:

    ::

        scene = QGraphicsScene()
        scene.addText("Hello, world!")

        view = QGraphicsView(scene)
        view.show()


    You can explicitly scroll to any position on the scene by using the scroll bars, or by calling :meth:`PySide.QtGui.QGraphicsView.centerOn` . By passing a point to :meth:`PySide.QtGui.QGraphicsView.centerOn` , :class:`PySide.QtGui.QGraphicsView` will scroll its viewport to ensure that the point is centered in the view. An overload is provided for scrolling to a :class:`PySide.QtGui.QGraphicsItem` , in which case :class:`PySide.QtGui.QGraphicsView` will see to that the center of the item is centered in the view. If all you want is to ensure that a certain area is visible, (but not necessarily centered,) you can call :meth:`PySide.QtGui.QGraphicsView.ensureVisible` instead.

    :class:`PySide.QtGui.QGraphicsView` can be used to visualize a whole scene, or only parts of it. The visualized area is by default detected automatically when the view is displayed for the first time (by calling :meth:`QGraphicsScene.itemsBoundingRect()<PySide.QtGui.QGraphicsScene.itemsBoundingRect>` ). To set the visualized area rectangle yourself, you can call :meth:`PySide.QtGui.QGraphicsView.setSceneRect` . This will adjust the scroll bars' ranges appropriately. Note that although the scene supports a virtually unlimited size, the range of the scroll bars will never exceed the range of an integer (INT\_MIN, INT\_MAX).

    :class:`PySide.QtGui.QGraphicsView` visualizes the scene by calling :meth:`PySide.QtGui.QGraphicsView.render` . By default, the items are drawn onto the viewport by using a regular :class:`PySide.QtGui.QPainter` , and using default render hints. To change the default render hints that :class:`PySide.QtGui.QGraphicsView` passes to :class:`PySide.QtGui.QPainter` when painting items, you can call :meth:`PySide.QtGui.QGraphicsView.setRenderHints` .

    By default, :class:`PySide.QtGui.QGraphicsView` provides a regular :class:`PySide.QtGui.QWidget` for the viewport widget. You can access this widget by calling :meth:`PySide.QtGui.QAbstractScrollArea.viewport` , or you can replace it by calling :meth:`PySide.QtGui.QAbstractScrollArea.setViewport` . To render using OpenGL, simply call setViewport(new :class:`PySide.QtOpenGL.QGLWidget` ). :class:`PySide.QtGui.QGraphicsView` takes ownership of the viewport widget.

    :class:`PySide.QtGui.QGraphicsView` supports affine transformations, using :class:`PySide.QtGui.QTransform` . You can either pass a matrix to :meth:`PySide.QtGui.QGraphicsView.setTransform` , or you can call one of the convenience functions :meth:`PySide.QtGui.QGraphicsView.rotate` , :meth:`PySide.QtGui.QGraphicsView.scale` , :meth:`PySide.QtGui.QGraphicsView.translate` or :meth:`PySide.QtGui.QGraphicsView.shear` . The most two common transformations are scaling, which is used to implement zooming, and rotation. :class:`PySide.QtGui.QGraphicsView` keeps the center of the view fixed during a transformation. Because of the scene alignment (setAligment()), translating the view will have no visual impact.

    You can interact with the items on the scene by using the mouse and keyboard. :class:`PySide.QtGui.QGraphicsView` translates the mouse and key events into *scene* events, (events that inherit :class:`PySide.QtGui.QGraphicsSceneEvent` ,), and forward them to the visualized scene. In the end, it's the individual item that handles the events and reacts to them. For example, if you click on a selectable item, the item will typically let the scene know that it has been selected, and it will also redraw itself to display a selection rectangle. Similiary, if you click and drag the mouse to move a movable item, it's the item that handles the mouse moves and moves itself. Item interaction is enabled by default, and you can toggle it by calling :meth:`PySide.QtGui.QGraphicsView.setInteractive` .

    You can also provide your own custom scene interaction, by creating a subclass of :class:`PySide.QtGui.QGraphicsView` , and reimplementing the mouse and key event handlers. To simplify how you programmatically interact with items in the view, :class:`PySide.QtGui.QGraphicsView` provides the mapping functions :meth:`PySide.QtGui.QGraphicsView.mapToScene` and :meth:`PySide.QtGui.QGraphicsView.mapFromScene` , and the item accessors :meth:`PySide.QtGui.QGraphicsView.items` and :meth:`PySide.QtGui.QGraphicsView.itemAt` . These functions allow you to map points, rectangles, polygons and paths between view coordinates and scene coordinates, and to find items on the scene using view coordinates.

.. method:: PySide.QtGui.QGraphicsView.alignment()


    :rtype: :class:`PySide.QtCore.Qt.Alignment`




    This property holds the alignment of the scene in the view when the whole scene is visible..

    If the whole scene is visible in the view, (i.e., there are no visible scroll bars,) the view's alignment will decide where the scene will be rendered in the view. For example, if the alignment is :attr:`Qt.AlignCenter<Qt.AlignmentFlag>` , which is default, the scene will be centered in the view, and if the alignment is ( :attr:`Qt.AlignLeft<Qt.AlignmentFlag>` | :attr:`Qt.AlignTop<Qt.AlignmentFlag>` ), the scene will be rendered in the top-left corner of the view.



.. method:: PySide.QtGui.QGraphicsView.backgroundBrush()


    :rtype: :class:`PySide.QtGui.QBrush`




    This property holds the background brush of the scene..

    This property sets the background brush for the scene in this view. It is used to override the scene's own background, and defines the behavior of :meth:`PySide.QtGui.QGraphicsView.drawBackground` . To provide custom background drawing for this view, you can reimplement :meth:`PySide.QtGui.QGraphicsView.drawBackground` instead.

    By default, this property contains a brush with the :attr:`Qt.NoBrush<Qt.BrushStyle>` pattern.

    .. seealso::  :meth:`QGraphicsScene.backgroundBrush<PySide.QtGui.QGraphicsScene.backgroundBrush>`  :meth:`PySide.QtGui.QGraphicsView.foregroundBrush` 


.. method:: PySide.QtGui.QGraphicsView.cacheMode()


    :rtype: :class:`PySide.QtGui.QGraphicsView.CacheMode`




    This property holds which parts of the view are cached.

    :class:`PySide.QtGui.QGraphicsView` can cache pre-rendered content in a :class:`PySide.QtGui.QPixmap` , which is then drawn onto the viewport. The purpose of such caching is to speed up the total rendering time for areas that are slow to render. Texture, gradient and alpha blended backgrounds, for example, can be notibly slow to render; especially with a transformed view. The :attr:`CacheBackground<QGraphicsView.CacheModeFlag>` flag enables caching of the view's background. For example:

    ::

        view = QGraphicsView()
        view.setBackgroundBrush(QImage(":/images/backgroundtile.png"))
        view.setCacheMode(QGraphicsView.CacheBackground)


    The cache is invalidated every time the view is transformed. However, when scrolling, only partial invalidation is required.

    By default, nothing is cached.

    .. seealso::  :meth:`PySide.QtGui.QGraphicsView.resetCachedContent`  :class:`PySide.QtGui.QPixmapCache`

.. method:: PySide.QtGui.QGraphicsView.centerOn(pos)


    :param pos: :class:`PySide.QtCore.QPointF`




    Scrolls the contents of the viewport to ensure that the scene coordinate ``pos`` , is centered in the view.

    Because ``pos`` is a floating point coordinate, and the scroll bars operate on integer coordinates, the centering is only an approximation.

    .. note::  If the item is close to or outside the border, it will be visible in the view, but not centered.

    .. seealso::  :meth:`PySide.QtGui.QGraphicsView.ensureVisible` 


.. method:: PySide.QtGui.QGraphicsView.dragMode()


    :rtype: :attr:`PySide.QtGui.QGraphicsView.DragMode`




    This property holds the behavior for dragging the mouse over the scene while the left mouse button is pressed..

    This property defines what should happen when the user clicks on the scene background and drags the mouse (e.g., scrolling the viewport contents using a pointing hand cursor, or selecting multiple items with a rubber band). The default value, :attr:`NoDrag<QGraphicsView.DragMode>` , does nothing.

    This behavior only affects mouse clicks that are not handled by any item. You can define a custom behavior by creating a subclass of :class:`PySide.QtGui.QGraphicsView` and reimplementing :meth:`PySide.QtGui.QGraphicsView.mouseMoveEvent` .



.. method:: PySide.QtGui.QGraphicsView.drawBackground(painter, rect)


    :param painter: :class:`PySide.QtGui.QPainter`
    :param rect: :class:`PySide.QtCore.QRectF`




    Draws the background of the scene using ``painter`` , before any items and the foreground are drawn. Reimplement this function to provide a custom background for this view.

    If all you want is to define a color, texture or gradient for the background, you can call :meth:`PySide.QtGui.QGraphicsView.setBackgroundBrush` instead.

    All painting is done in *scene* coordinates. ``rect`` is the exposed rectangle.

    The default implementation fills ``rect`` using the view's :meth:`PySide.QtGui.QGraphicsView.backgroundBrush` . If no such brush is defined (the default), the scene's :meth:`PySide.QtGui.QGraphicsView.drawBackground` function is called instead.

.. method:: PySide.QtGui.QGraphicsView.setCacheMode(mode)


    :param mode: :class:`PySide.QtGui.QGraphicsView.CacheMode`




    This property holds which parts of the view are cached.

    :class:`PySide.QtGui.QGraphicsView` can cache pre-rendered content in a :class:`PySide.QtGui.QPixmap` , which is then drawn onto the viewport. The purpose of such caching is to speed up the total rendering time for areas that are slow to render. Texture, gradient and alpha blended backgrounds, for example, can be notibly slow to render; especially with a transformed view. The :attr:`CacheBackground<QGraphicsView.CacheModeFlag>` flag enables caching of the view's background. For example:

    ::

        view = QGraphicsView()
        view.setBackgroundBrush(QImage(":/images/backgroundtile.png"))
        view.setCacheMode(QGraphicsView.CacheBackground)


    The cache is invalidated every time the view is transformed. However, when scrolling, only partial invalidation is required.

    By default, nothing is cached.

    .. seealso::  :meth:`PySide.QtGui.QGraphicsView.resetCachedContent`  :class:`PySide.QtGui.QPixmapCache`

    :param matrix: :class:`PySide.QtGui.QMatrix`
    :param combine: :class:`PySide.QtCore.bool`




    Sets the view's current transformation matrix to ``matrix`` .

    If ``combine`` is true, then ``matrix`` is combined with the current matrix; otherwise, ``matrix``\ *replaces* the current matrix. ``combine`` is false by default.

    The transformation matrix tranforms the scene into view coordinates. Using the default transformation, provided by the identity matrix, one pixel in the view represents one unit in the scene (e.g., a 10x10 rectangular item is drawn using 10x10 pixels in the view). If a 2x2 scaling matrix is applied, the scene will be drawn in 1:2 (e.g., a 10x10 rectangular item is then drawn using 20x20 pixels in the view).

    Example:

    ::

        scene = QGraphicsScene()
        scene.addText("GraphicsView rotated clockwise")

        view = QGraphicsView(scene)
        view.rotate(90) # the text is rendered with a 90 degree clockwise rotation
        view.show()


    To simplify interation with items using a transformed view, :class:`PySide.QtGui.QGraphicsView` provides mapTo... and mapFrom... functions that can translate between scene and view coordinates. For example, you can call :meth:`PySide.QtGui.QGraphicsView.mapToScene` to map a view coordinate to a floating point scene coordinate, or :meth:`PySide.QtGui.QGraphicsView.mapFromScene` to map from floating point scene coordinates to view coordinates.

.. method:: PySide.QtGui.QGraphicsView.setSceneRect(rect)

    This property holds the area of the scene visualized by this view..

    The scene rectangle defines the extent of the scene, and in the view's case, this means the area of the scene that you can navigate using the scroll bars.

    If unset, or if a null :class:`PySide.QtCore.QRectF` is set, this property has the same value as :meth:`QGraphicsScene.sceneRect<PySide.QtGui.QGraphicsScene.sceneRect>` , and it changes with :meth:`QGraphicsScene.sceneRect<PySide.QtGui.QGraphicsScene.sceneRect>` .
